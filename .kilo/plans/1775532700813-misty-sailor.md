# Plan: Systematic Evolution of KV/Skip Idea into Publishable Variants

## Goal
Evolve the current layer skipping implementation (v0-v2) into advanced, research-grade variants (v3-v7) suitable for publication, focusing on residual safety, token-level routing, KV-awareness, and planning-ahead mechanisms.

## Current State Analysis
- **v0 (Static Layer Skipping)**: Implemented (`run_skip`), tested, provides baseline (loss ↑, speed ↑)
- **v1 (Heuristic Dynamic Skipping)**: Implemented (`run_entropy`), tested, noisy thresholds, unstable
- **v2 (Learned VoC)**: Implemented (`run_voc`), current focus - trains VoC scorer and Router, threshold-based decisions, myopic (no future awareness)

## Implementation Roadmap

### v2 Refinements (Current → Production-Ready)
- **Calibrate thresholds**: Implement adaptive threshold selection using validation set loss minimization
- **Per-layer normalization**: Add layer-specific feature normalization for better generalization
- **Router improvements**: Increase capacity (deeper network), add dropout, implement early stopping
- **Feature engineering**: Experiment with additional signals (gradient norms, attention sparsity)
- **Evaluation**: Compare against v0/v1 baselines, measure stability across different prompts

### v3: Residual-Safe Skipping (Critical Fix)
- **Transformer Math Respect**: Ensure skips preserve LN → Attn → Residual and LN → MLP → Residual connections
- **Identity Residual Implementation**: When skipping, return input hidden_states unchanged (already partially done, verify correctness)
- **Importance Recalculation**: Update importance metric to account for residual contributions properly
- **Stability Testing**: Verify no gradient explosions, consistent loss scaling with skip rate
- **HuggingFace Compatibility**: Ensure patch works with different model architectures

### v4: Token-Level Routing
- **Per-Token Decisions**: Instead of whole-layer skip, route individual unimportant tokens
- **Attention Masking**: Modify attention computation to skip tokens only in attention/MLP
- **Token Importance Scoring**: Extend Router to output per-token scores
- **Implementation Strategy**:
  - Modify VoCSkipLayer to handle token-level routing
  - Update attention layers to accept token masks
  - Maintain KV cache compatibility
- **Quality-Speed Tradeoff**: Expect better perplexity retention at same compute reduction

### v5: KV-Aware Importance
- **Attention Statistics Integration**: Incorporate attention weights, key/query norms, value variances
- **KV Cache Analysis**: Use cached KV states to predict future attention patterns
- **Extended Features**: Add KV-based features to Router input
- **Future Usefulness Scoring**: Score based on "will this token be attended later?"
- **Implementation**: Hook into attention layers to collect statistics, integrate with Router training

### v6: Planning-Aware (Research-Grade)
- **Multi-Step Prediction**: Train critic to predict usefulness 5+ steps ahead
- **Temporal Consistency**: Use sequence-level patterns for better decisions
- **RL Integration**: Implement reinforcement learning for optimal skip policies
- **Novelty**: This enters NeurIPS-level territory with planning-ahead mechanisms
- **Components**: 
  - Sequence predictor model
  - Temporal feature aggregation
  - RL training loop

### v7: Joint System Architecture
- **Planner Module**: Importance predictor with multi-step awareness
- **Executor Module**: Handles skip/route decisions and execution
- **KV Controller**: Manages cache eviction/compression policies
- **Closed-Loop Feedback**: Use loss/attention drift to refine decisions
- **Full Integration**: Combine all components into unified system

## Metrics to Track
- **Loss/Perplexity**: Primary quality metric, track degradation with skip rate
- **Tokens/Sec**: Speed improvement measurement
- **% Compute Skipped**: Efficiency metric, correlate with quality loss
- **Attention Drift**: Measure how much attention patterns change with skipping
- **Stability**: Gradient norms, training convergence, robustness to different inputs

## Research Hooks & Publication Angles
- **v4 (Token Routing)**: Novel per-token skipping, potential for high-impact paper
- **v5 (KV-Aware)**: Attention-aware efficiency, good for ML sys conferences
- **v6 (Planning)**: Multi-step prediction, strong theoretical contribution
- **v7 (Full System)**: Integrated architecture, comprehensive solution

## Implementation Order & Dependencies
1. **v2 → v2-refined**: Stabilize current approach
2. **v3**: Fix residual safety (prerequisite for all advanced variants)
3. **v4**: Token-level routing (big quality gain)
4. **v5**: KV integration (complementary to v4)
5. **v6**: Planning (builds on v4/v5)
6. **v7**: System integration (capstone)

## Technical Considerations
- **HuggingFace Compatibility**: Ensure all changes are non-breaking patches
- **KV Cache Handling**: Maintain compatibility with existing caching mechanisms
- **Memory Efficiency**: Monitor memory usage with per-token operations
- **Training Complexity**: Manage increased training time for advanced models
- **Evaluation Rigor**: Use proper validation sets, statistical significance testing

## Success Criteria
- Each variant shows Pareto improvement over baselines
- Stable training and inference across different model sizes
- Code quality suitable for open-source release
